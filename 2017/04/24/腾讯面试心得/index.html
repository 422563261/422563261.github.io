发表于 2017-04-24   |

1.负载均衡一般几个服务器比较好？

       负载均衡（Load Balance）是分布式系统架构设计中必须考虑的因素之一，它通常是指，将请求/数据均匀分摊到多个操作单元上执行，负载均衡的关键在于均匀。常见互联网分布式架构如上，分为客户端层、反向代理nginx层、站点层、服务层、数据层。

总结：我应该是记错了问题，并没查到相关的资料，我的认为是如果服务器没有饱和，就没必要实现负载均衡。

2.怎么消除DOM太多引起的卡顿？

网页卡顿，有DOM操作的一份功劳，解决办法：

一、合并多次的DOM操作为单次的DOM操作

未优化前：

element.style.borderColor = '#f00';
element.style.borderStyle = 'solid';
element.style.borderWidth = '1px';

优化后：

// 优化方案1
element.style.cssText += 'border: 1px solid #f00;';
// 优化方案2
element.className += 'empty';

个人经常用的是方案二，感觉维护性可读性都比较好，但性能上的损耗，因为它需要查询CSS类。方案一的优势可以动态传值。

二、把DOM元素离线或隐藏后修改

  • 使用文档片段

    var fragment = document.createDocumentFragment();
// 一些基于fragment的大量DOM操作
...
document.getElementById('myElement').appendChild(fragment);

  • 通过设置DOM元素的display样式为none来隐藏元素

           这种方式是通过隐藏页面的DOM元素，达到在页面中移除元素的效果，经过大量的DOM操作后恢复元素原来的display样式。对于这类会引起页面重绘或重排的操作，就只有隐藏和显示DOM元素这两个步骤了。

    var myElement = document.getElementById('myElement');
myElement.style.display = 'none';
// 一些基于myElement的大量DOM操作
...
myElement.style.display = 'block';

  • 克隆DOM元素到内存中

           这种方式是把页面上的DOM元素克隆一份到内存中，然后再在内存中操作克隆的元素，操作完成后使用此克隆元素替换页面中原来的DOM元素。这样一来，影响性能的操作就只是最后替换元素的这一步操作了，在内存中操作克隆元素不会引起页面上的性能损耗。

    var old = document.getElementById('myElement');
var clone = old.cloneNode(true);
// 一些基于clone的大量DOM操作
...
old.parentNode.replaceChild(clone, old);

三、设置具有动画效果的DOM元素的position属性为fixed或absolute

       把页面中具有动画效果的元素设置为绝对定位，使得元素脱离页面布局流，从而避免了页面频繁的重排，只涉及动画元素自身的重排了。

3.考虑过https会延迟的问题吗？

  • HTTPS链接和HTTP链接都建立在TCP协议之上。
  • HTTP链接比较单纯，使用三个握手数据包建立连接之后，就可以发送内容数据了。
  • 而HTTPS链接，它也采用TCP协议发送数据，所以它也需要三步握手过程。而且，在这三步结束以后，它还有一个SSL握手。

总结：所以，如果是对安全性要求不高的场合，为了提高网页性能，建议不要采用保密强度很高的数字证书。但我感觉，现在计算资源和网络资源都提高了很多，可以推广https。

参考资料引用：阮一峰老师的博客

4. 如何阻止POST数据被抓包？

       将要提交的参数先做加密，然后把加密的信息做一次md5摘要，也就是签名，然后把摘要连同参数一起回传给服务器，服务器拿到参数后，同样的方式加密做md5摘要，然后两个摘要做对比，如果不相等参数便是被篡改了，否则可信。这种方式已经是很成熟的url参数防篡改技术了，简单易用成本低，安全系数高，并且不会大大增加url的长度。

总结：被抓包感觉无可避免，但我们可以提高数据破解复杂度。

8. 如何实现广度优先和深度优先遍历？

深度优先：

let nodes = [];  
function deepTraversal(node, nodes) {  
    if (node !== null) {    
        nodes.push(node);    
        let children = [...node.children];  
        children.forEach(child => {
            deepTraversal(child, nodes);    
        })    
    }    
}

广度优先：

function wideTraversal(selectNode) {  
    let nodes = [];  
    if (selectNode !== null) {  
        let queue = [];  
        queue.unshift(selectNode);  
        while (queue.length !== 0) {  
            let item = queue.shift();  
            nodes.push(item);  
            let children = [...item.children];  
            children.forEach(child => {
                queue.push(child);
            }) 
        }  
    }  
    return nodes;  
}

9. 单页应用会导致内存泄漏，请问如何解决？

3种高强度操作：

a: 强交互应用中的ajax或者socket请求处理

b: setInterval, requestAnimationFrame等循环处理

c: click等高频处理函数

       内存泄漏问题中，bc的情况其实是很少的，主要致命的内存泄漏出现在a情况，因为大部分请求都可能伴随着页面的局部刷新，特别是引入第三方框架时（比如chart相关，highchart、d3等），仅仅remove相应dom一般不会释放第三方对象，这时候就需要手动进行destroy。

例：用vue包装的highchart组件需要在组件生命周期末端进行销毁

mounted() {
    this.initOptions();
    this.draw();
},
beforeDestroy() {
    this.myChart && this.myChart.destroy();
}

10. repaint（重绘）与reflow（重排），哪些情况会触发？

一、定义

reflow：对于DOM结构中的各个元素都有自己的盒子，这些都需要浏览器根据各种样式来计算，并根据计算结果将元素放到它该出现的位置。

repaint：当各种盒子的位置、大小以及其他属性，例如颜色、字体大小等都确定下来后，浏览器于是便把这些元素都按照各自的特性绘制了一遍。

注意：回流必将引起重绘，而重绘不一定会引起回流

二、因素

导致reflow因素：

  • 改变窗囗大小
  • 改变文字大小
  • 添加/删除样式表
  • 内容的改变，如用户在输入框中敲字
  • 激活伪类，如:hover (IE里是一个兄弟结点的伪类被激活)
  • 操作class属性
  • 脚本操作DOM
  • 计算offsetWidth和offsetHeight
  • 设置style属性

只会导致repaint因素：

  • 使用display:none;的方式隐藏一个结点会导致repaint与reflow，使用visibility:hidden;进行dom隐藏仅仅导致repaint
