半透明边框

background-clip

参数:

padding-box || border-box(默认值)

区别:

  • 默认状态下border-box状态下边框会入侵背景

  • padding-box状态下边框不会入侵背景

条纹背景

linear-gradient

参数

<方向,颜色 (位置)，颜色 (位置)>

方向

  1. 可以用deg表示，也可以用top,bottom,left,right表示

    to top -> 0deg
    to right -> 90deg
    to bottom -> 180deg
    to left -> -90deg(或270deg)
    to top left -> -45deg(或315deg)
    to top right -> 45deg
    to bottom left -> -135deg(或225deg)
    to bottom right -> 135deg

位置

  1. 位置可以省略，浏览器默认会把第一个颜色的位置设置为0%，把最后一个颜色的位置设置为100%。

    background-image: linear-gradient(red 0%,blue 100%);
//等价于上一个
background-image: linear-gradient(red,blue);

  2. 若渐变只有两种颜色，且第一个颜色的位置设置为n%，第二个颜色的位置设置为m%。则浏览器会将0%-n%的范围设置为第一个颜色的纯色，n%-m%的范围设置为第一个颜色到第二个颜色的过渡，m%-100%的范围设置为第二个颜色的纯色。

    background-image: linear-gradient(red 30%,blue 60%);
//等价于上一个
background-image: linear-gradient(red 0%,red 30%,blue 60%,blue 100%);

连续图像边框

实现下面的效果可以通过两个div来实现，如果只用一个呢？

思路:

感谢背景与边框(第三版)(http://w3.org/TR/css3-background)引入了对 CSS渐变和背景的扩展,使得我们只用一个元素就能达成完全一样的效果。主要的思路就是在石雕背景图片之上,再叠加一层纯白的实色背景。为 了让下层的图片背景透过边框区域显示出来,我们需要给两层背景指定不同 的 background-clip 值。最后一个要点在于,我们只能在多重背景的最底 层设置背景色,因此需要用一道从白色过渡到白色的 CSS 渐变来模拟出纯 白实色背景的效果。

padding: 1em;
border: 1em solid transparent; 
background: linear-gradient(white, white),
url(stone-art.jpg); 
background-size: cover;
background-clip: padding-box, border-box; 
background-origin: border-box;

原因是 background-origin 的 默认值是padding-box,因此,图片的显示尺寸不仅取决于padding box的 尺寸,而且被放置在了padding box的原点(左上角)。我们看到的实际上 就是背景图片以平铺的方式蔓延到border box区域的效果。为了修正这个问 题,只需把 background-origin 也设置为 border-box。

background-size: 10px 10px,50px 50px;

宿主元素应用position: relative样式,并为伪元素设置position: absolute,然后再把所有偏移量设置为零,以便让它在水平和垂直方向上都 被拉伸至宿主元素的尺寸.此时,用伪元素生成的方块是重叠在内容之上的,一旦给它设置背景,会把内容完全盖住。为了修复这个问题,我们可以给伪元素设置z-index:-1。

视觉效果

单侧投影

box-shadow

使用高斯模糊算法，将阴影半径进行模糊处理。这在本质上表示在阴影边缘发生阴影色和纯透明色之间的颜色过渡长度近似于模糊半径的两倍。

box-shadow: 0px 5px 4px #000;

使用 4px 的模糊半径意味着投影的尺寸会比元素本身的尺寸大约 8px, 因此投影的最外圈会从元素的四面向外显露出来。我们只需改变偏移量, 就可以把投影的顶部和左侧隐藏起来,只要这两个方向上的偏移量不小于 4px 就可以了。但是,这在某种程度上会导致外露的投影太过浓重,看起来不是很美观。

最终的解决方案来自box-shadow第四个长度参数。它排在模糊半径参数之后,称作扩张半径。这个参数会根据你指定的值去扩大或 (当指定负值时)缩小投影的尺寸。如果我们应用一个负的扩张半径,而它的值刚好等于模糊半径,那么投影的尺寸就会与投影所属元素的尺寸完全一致。除非用偏移量(前两个长度参数)来移动它,我们将完全看不见任何投影。

box-shadow: 0px 5px 4px -4px #000;

实现这样的效果，我们同样要运用之前的::before伪类，因为直接在元素上运用blur()会导致文字一起毛玻璃化。

    //background参数，第一个url，第二个background:position，第三个background:attachment。

.main {
    position: relative;

    overflow: hidden;
}
.main::before { 
    content: '';
    position: absolute;
    top: 0; right: 0; bottom: 0; left: 0; 
    filter: blur(20px);
    margin: -30px;
    z-index:-1;
}
//这里搭配overflow和margin原因是因为为了保证.main完整被模糊化，也保证模糊化超出的部分能够被裁剪掉

结构与布局

自适应内部元素

在CSS3的世界里，width属性又多了几个关键字成员，fill-available,max-content, min-content, 以及fit-content
大神讲解

1. 理解width:fill-available

width:fill-available比较好理解，比方说，我们在页面中扔一个没有其他样式的<div>元素，则，此时，该<div>元素的width表现就是fill-available自动填满剩余的空间。也就是我们平常所说的盒模型的margin,border,padding的尺寸填充。

出现fill-available关键字值的价值在于，我们可以让元素的100%自动填充特性不仅仅在block水平元素上，其他元素，例如，我们一直认为的包裹收缩的inline-block元素上：

div { display:inline-block; width:fill-available; }

此时，元素兼具了块状元素的自动填充特性以及内联元素的定位对齐等特性。于是，我们就可以直接使用line-height让一个块状表现的元素垂直居中。

2. 理解width:max-content

max-content的行为表现可以这么理解，假设我们的容器有足够的宽度，足够的空间，此时，所占据的宽度是就是max-content所表示的尺寸。

这个不是很好解释，还是看大神的解释吧

3. 理解width:min-content

min-content宽度表示的并不是内部那个宽度小就是那个宽度，而是，采用内部元素最小宽度值最大的那个元素的宽度作为最终容器的宽度。

4. 理解width:fit-content

width:fit-content和CSS2.1中的float, absolute, inline-block的尺寸收缩表现是一样的。

就拿水平居中效果举例，首先浮动肯定不行，因为只有左浮动和右浮动；绝对定位压根不占据空间，普通流中根本无法应用，而inline-block需要父级使用text-align:center，而本身可能还需要text-align:left略烦。

而width:fit-content可以没有这些烦恼，因为，width:fit-content可以实现元素收缩效果的同时，保持原本的block水平状态，于是，就可以直接使用margin:auto实现元素向内自适应同时的居中效果了。

垂直居中（网页居中）

实现这样的效果我们一般是这样实现的：

----html----
<div class="box-center"></div>

----css----

方法一：
.box-center{
    width:200px;
    height:200px;
    position:absolute;
    top:0;left:0;right:0;bottom:0;
    margin:0;
}

方法二：
.box-center{
    width:200px;
    height:200px;
    position:absolute;
    top:50%;left:50%;
    margin-left:-100px;
    margin-top:-100px;
}

方法三：
.box-center{
    width:200px;
    height:200px;
    position:absolute;
    top:calc(50% - 100px);
    left:calc(50% - 100px);
}

方法四：
.box-center{
    width:200px;
    height:200px;
    margin: 50vh auto 0;
    transform:translateY(-50%);
}

视口相关的长度单位(方法四)

  1. vw是与视口宽度相关的。与常人的直觉不符的是，1vw 实际上表示视口宽度的 1%，而不是 100%。
  2. 与 vw 类似，1vh 表示视口高度的 1%。
  3. 当视口宽度小于高度时，1vmin 等于 1vw，否则等于 1vh。
  4. 当视口宽度大于高度时，1vmax 等于 1vw，否则等于 1vh。

以上的方法都有一个共同的弊端，那就是必须设置width和height，通常来说，需要居中的对象的大小应该由内容决定。

方法五：
.box-center{
    position:absolute;
    top:50%;left:50%;
    transform:translate(-50%,-50%);
}

translate()变形函数中使用百分比值时，是以这个元素自身的宽度和高度为基准进行换算和移动的，而这正式我们所需要的。

  • 我们有时不能选用绝对定位，因为它对整个布局的影响太过强烈。
  • 如果需要居中的元素已经在高度上超过了视口，那它的顶部会被视口裁切掉。
  • 在某些浏览器中，这个方法可能会导致元素的显示有一些模糊，因为元素可能被放置在半个像素上。这个问题可以用 transform- style: preserve-3d来修复。



基于flexbox的解决方法是最优解：

方法六：
body{
    display:flex;
    min-height:100vh;
    margin:0;
}
.box-center{
    margin:auto;
}

紧贴底部的页脚

我们经常会遇到这个问题，需要footer紧贴底部，在页面长度足够的时候可以实现，但是页面内容不够的时候，就会产生上图一样的效果。

----css----
//if header.height=="25px"
//if footer.height=="70px"

main{
    min-height:calc(100% - 25px - 70px);
    box-sizing:border-box;
}

----html----
<header>
    <h1>this is title</h1>
</header>
<main>
    <p>this si content</p>
</main>
<footer>
    <p>this is footer</p>
</footer>

上面列出的是一种简单的解决办法，适用于简单布局。一旦布局复杂了之后，就不适用了。

下面将给出最优解，同样是基于flexbox：

----css----
body{
    display:flex;
    flex-flow:colum;
    min-height:100vh;
}
main{
    flex:1;
}

设置flex-flow:colum;是为了让元素垂直排列，min-height:100vh;是为了让body至少会占据整个视口的高度。flex:1;可以自动伸展并沾满所有可用空间。

.callout{
    transform-origin: 1.4px -.4px;
    transition: .5s cubic-bezier(.25,.1,.3,1.5) transform;
}

----html----

<input type="text">
<div class="callout">233</div>

解释一下：

cubic-bezier(.25,.1,.3,1.5)的效果：

在50%（250ms）的时候已经达到100变形程度。70%（350ms）的时候达到巅峰110%。失去焦点的时候为什么采用transition: .25s transform?因为如果还是采用上面的函数，则会在350ms达到110%峰值scale(-0.1)，这不是我们想看到的。采用默认ease函数，而ease函数从0~100%需要500ms，在感觉上会比之前慢一半。所以时间采用0.25s。

逐帧动画

GIF的缺点：

  • 颜色被限制在256
  • 不具有透明性
  • 无法修改GIF参数，不具有易维护性

不使用GIF实现下列的动画效果，我们采用APNG（逐帧动画）：

----css----
.loader{
    width:100px;
    height:100px;
    background:url("loader.png") 0 0;
    animation:loader 1s infinite step(8);

    /*隐藏文本*/
    overflow:hidden;
    text-indent:200%;
    white-space:nowrap;
}
@keyframes loader{
    to{
        background-position:-800px 0;
    }
}

----html----
<div class="loader">loading</div>

首先我们需要一个容器，设定宽度和高度，然后在里面填一些文字（确保可访问性）。与贝塞尔曲线不同的是，step()会根据你指定的步量，把整个动画切分为多帧，而且在整个动画中会在帧与帧之间硬切，不会做任何平滑过渡处理。

打字动画

能够完美实现打字动画的一个重要因素就是使用“等宽字体”。

----css----
body{
    font-family:Consolas;
}
.test{
    width: 12ch;
    overflow: hidden;
    white-space: nowrap;//阻止文本折行
    animation: typing 3s steps(12),
                carcet 1s steps(1) infinite;
    border-right: 1px solid transparent;

}
@keyframes typing{
    from{width: 0;}
}
@keyframes carcet{
    50%{
        border-color:currentColor; //与color颜色一致
    }
}

ch 字符相对单位

注释：此单位，必须与等宽字体一起使用才有用。一个字符长度为1ch。

这里有一个难度点。我们首先看一个图：

根据观察函数图，得出颜色值的切换只会发生在动画周期的末尾。

@keyframes carcet{
    to {
        border-color:current;
    }
}

若按照上面这种书写方式，我们将看到透明的边框占99.999……%的时间，最后那可以忽略不记的时间才变成黑色边框，肉眼根本观察不到。所以设置50%中心点。

沿环形路径平移的动画

----css----
.circle{
    width: 200px;
    height: 200px;
    border-radius: 50%;
    background: yellow;
    position: relative;
}
.box{
    width: 20px;
    height: 20px;
    overflow: hidden;
    border-radius: 50%;
    position: absolute;
    left: 90px;
    animation: move 6s infinite linear;
    transform-origin: 50% 100px;
    //调整旋转中心至距离顶部中心R的点（圆心）
}
.panda{
    width: 20px;
    height: 20px;
    animation: inherit;
    animation-direction: reverse;//与box动画相反
}
@keyframes move{
    to{
        transform: rotate(1turn);//1turn=360deg
    }
}
